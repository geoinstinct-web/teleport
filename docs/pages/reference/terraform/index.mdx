The Teleport Terraform provider allows Terraform users to configure Teleport
from Terraform.

This section is the Teleport Terraform Provider reference.
It lists all the supported resources and their fields.

<Admonition type="tip">
To get started with the Terraform provider, you must start with the installation guide.
Once you got a working provider, we recommend you to follow the
["Managing users and roles with IaC"](../management/dynamic-resources/user-and-role.mdx) guide.
</Admonition>

The provider exposes Teleport resources both as Terraform datasources and Terraform resources.
Datasources are used by Terraform to fetch information from Teleport, while resources are used
to create resources in Teleport.

- [list of supported datasources](./datasources/index.mdx)
- [list of supported resources](./resources/index.mdx)

## Example Usage

```hcl
terraform {
  required_providers {
    teleport = {
      source  = "terraform.releases.teleport.dev/gravitational/teleport"
      version = "~> 15.0"
    }
  }
}

provider "teleport" {
  # Update addr to point to Teleport Auth/Proxy
  # addr              = "auth.example.com:3025"
  addr               = "proxy.example.com:443"
  identity_file_path = "terraform-identity/identity"
}
```

## Schema

### Optional

- `addr` (String) host:port where Teleport Auth server is running.
- `cert_base64` (String) Base64 encoded TLS auth certificate.
- `cert_path` (String) Path to Teleport auth certificate file.
- `dial_timeout_duration` (String) DialTimeout sets timeout when trying to connect to the server.
- `identity_file` (String, Sensitive) Teleport identity file content.
- `identity_file_base64` (String, Sensitive) Teleport identity file content base64 encoded.
- `identity_file_path` (String) Teleport identity file path.
- `key_base64` (String, Sensitive) Base64 encoded TLS auth key.
- `key_path` (String) Path to Teleport auth key file.
- `profile_dir` (String) Teleport profile path.
- `profile_name` (String) Teleport profile name.
- `retry_base_duration` (String) Retry algorithm when the API returns 'not found': base duration between retries (https://pkg.go.dev/time#ParseDuration).
- `retry_cap_duration` (String) Retry algorithm when the API returns 'not found': max duration between retries (https://pkg.go.dev/time#ParseDuration).
- `retry_max_tries` (String) Retry algorithm when the API returns 'not found': max tries.
- `root_ca_base64` (String) Base64 encoded Root CA.
- `root_ca_path` (String) Path to Teleport Root CA.
